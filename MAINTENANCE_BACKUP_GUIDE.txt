SARVAM DENTAL CLINIC - POST LAUNCH MAINTENANCE + BACKUP GUIDE
=============================================================

1) SOURCE OF TRUTH
------------------
- GitHub repository:
  https://github.com/rameshdhv/svc
- Production site:
  https://sarvamdentalclinic.netlify.app/
- Supabase project
- Netlify site configuration

Keep GitHub as the primary code source of truth.

2) WHAT TO BACK UP
------------------
Repository and docs:
- README.md
- HANDSOFF.md
- DENTIST_APP_REBUILD_BLUEPRINT.txt
- SUPABASE_MIGRATION_GUIDE.txt
- HOSTING_DEPLOYMENT_GUIDE.txt
- FEATURE_CODENAMES.txt
- supabase/schema.sql
- netlify.toml

Secrets and platform settings:
- SUPABASE_URL
- SUPABASE_ANON_KEY
- SUPABASE_SERVER_KEY
- SUPABASE_SERVICE_ROLE_KEY only if legacy fallback is still used
- Netlify site settings
- Supabase Auth URL Configuration

Optional future WhatsApp data if Twilio is enabled later:
- TWILIO_ACCOUNT_SID
- TWILIO_AUTH_TOKEN
- TWILIO_WHATSAPP_FROM

3) SAFE CHANGE WORKFLOW
-----------------------
1. pull latest code
2. run local dev
3. verify booking and admin flows
4. make change
5. test on desktop and mobile
6. push to GitHub
7. let Netlify deploy
8. smoke test production

Preferred local run:
- npm install
- npx netlify dev --port 8889 --staticServerPort 4002

Alternative if ports are free:
- npx netlify dev --port 8888 --staticServerPort 4001

4) SMOKE TEST AFTER ANY CHANGE
------------------------------
Public:
- home, services, gallery load
- booking request works
- same-day expired slots hide
- weekly off overlay works
- blocked date/session behavior works

Admin:
- dentist login
- staff login
- TOTP flow
- forgot password / reset password
- Day Snapshot
- approve / reject
- Book for Patient
- Control Panel open/close on desktop and mobile
- Control Panel Back-button close behavior on mobile/browser
- Manage Users
- Current Access Users
- analytics page
- Back to Dashboard visible only for logged-in admins

5) IF DENTIST REQUESTS CHANGES
------------------------------
Classify the request first:
- UI only
- workflow logic
- auth/security
- schema/data
- hosting/deployment

Then ask:
- all devices or one device only?
- will it affect live appointments?
- does it require backend changes?

6) MONTHLY / PERIODIC TASKS
---------------------------
Monthly:
- export critical Supabase data if business wants backups
- verify active users list
- verify dentist/staff accounts still correct
- verify reset-password links still route correctly

When auth or hosting changes:
- recheck Supabase Auth URL Configuration
- recheck Netlify env vars
- trigger fresh deploy

7) INCIDENT RECOVERY
--------------------
If something breaks in production:
1. roll back to previous Netlify deploy
2. revert latest Git commit if needed
3. verify env vars were not changed incorrectly
4. retest booking + admin flows

8) SECURITY PRACTICES
---------------------
- never expose server key in frontend code
- keep backend-only actions in serverless/API layer
- do not rely on hidden links for security; auth and role checks are the real protection
- keep dentist/staff accounts limited to real users
- if an old test auth user cannot be deleted cleanly, banning the user is acceptable until a later cleanup

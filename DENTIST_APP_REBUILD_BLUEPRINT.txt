Product Scope
-------------

   - gallery page for before/after treatments

Public Booking Rules
--------------------
- Slot interval controlled by settings: 30 or 60 minutes.
- Booking window limited to next N days from settings.
- Same-day expired slots hidden automatically.
- Approved slots unavailable.
- Weekly off closes booking.
- Blocked date ranges close booking.
- Blocked sessions close booking.
- Full-day unavailable state uses centered yellow overlay over blurred disabled slots.
- Successful booking stays on selected date and shows centered modal.

Admin Login Rules
-----------------
- Role switch UI: Login as Dentist / Login as Staff.
- Standard login flow: email + password + authenticator code.
- TOTP setup required for admin access.
- Forgot-password uses email recovery link.
- Recovery link should lead to dedicated reset form before normal dashboard flow resumes.
- Resend Confirmation Link should live in email-related recovery states, not on the base login screen.

Dashboard Rules
---------------
- Appointment lifecycle:
  Pending -> Approved / Rejected -> Completed / No-show
- Approved future appointments can be unblocked.
- Action notes must record actor role, for example:
  - Booked by staff
  - Approved by dentist
  - Rejected by staff
  - Unblocked by dentist: customer requested cancellation
- Day Snapshot is collapsible.
- Appointments section is collapsible.
- Emergency Session Blocking is collapsible.
- Holiday / Leave Blocking is collapsible.
- On mobile, Day Snapshot open by default, other collapsible dashboard sections closed by default.

Control Panel Rules
-------------------
- Visible label: Control Panel.
- Dashboard button text should also be Control Panel.
- Modal must scroll internally on all devices.
- Modal must have top-right close button.
- Browser/mobile Back should close Control Panel before leaving dashboard.
- Staff must not see Reset Past Data.

Control Panel Sections
----------------------
- slot duration
- effective from date
- show slots for next N days
- permanent weekly off
- Manage Users
- Current Access Users
- Reset Past Data (dentist only)

Manage Users
------------
- Dentist can create new dashboard users without opening Supabase.
- Add-user flow uses secure backend route.
- Add-user requires fresh TOTP confirmation before user creation.
- Show Current Access Users list with:
  - total count
  - full name
  - email
  - role

WhatsApp Behavior
-----------------
- Current implementation uses redirect-based WhatsApp drafts, not API delivery verification.
- State changes finalize only after user confirms Message Sent.
- This guarded behavior applies to:
  - approve / reject notify
  - Book for Patient notify
  - conflict-driven cancellation before session block
  - customer requested cancellation during unblock

Conflict and Blocking Rules
---------------------------
- Weekly holidays cannot save if future Pending/Approved appointments exist on those weekdays.
- Session blocking supports multi-select through modal slot picker.
- If approved bookings already exist in a slot being blocked, warn first.
- Proceed-anyway path must notify affected patient(s) before finalizing cancellation and block.
- Manual booking conflicts must show centered popup and distinguish:
  - already booked slot
  - blocked slot or blocked day with reason
- Unblock reasons are fixed options:
  - customer didnt showup
  - customer requested cancellation

Realtime and Alerts
-------------------
- When admin dashboard is open and a new booking arrives:
  - show bottom-left alert
  - play short sound
  - refresh data
- Use Supabase realtime if available.
- Keep polling fallback for reliability.

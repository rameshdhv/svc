HOSTING DEPLOYMENT GUIDE
========================

Purpose
-------
Use this guide when deploying Sarvam Dental Clinic to Netlify, Vercel, Hostinger, GoDaddy, or another host.

Project Architecture
--------------------
This project is:
- static frontend (HTML, CSS, vanilla JS)
- Supabase backend for auth and data
- small secure backend routes for high-privilege admin actions

What Works On Static Hosting Alone
----------------------------------
These parts can run on static hosting with Supabase browser access:
- public website
- gallery
- public booking requests
- admin login UI
- admin dashboard reads and most normal dashboard actions that use Supabase browser client
- analytics page

What Requires Secure Backend Execution
--------------------------------------
These features require server-side execution and a backend key:
- Manage Users -> Add User
- Manage Users -> Current Access Users
- Reset Past Data

If the host cannot run secure backend code, those features will not work safely.

Required Environment Variables
------------------------------
Frontend/runtime:
- SUPABASE_URL
- SUPABASE_ANON_KEY

Server-side:
- SUPABASE_SERVER_KEY preferred
- SUPABASE_SERVICE_ROLE_KEY legacy fallback

Optional future WhatsApp API env vars:
- TWILIO_ACCOUNT_SID
- TWILIO_AUTH_TOKEN
- TWILIO_WHATSAPP_FROM

Netlify
-------
Best fit for the current project.

Why:
- `netlify.toml` already exists
- backend functions already exist in `netlify/functions`
- frontend already calls `/api/*`

Current server routes:
- /api/admin-create-user
- /api/admin-list-users
- /api/reset-past-data

Netlify setup steps:
1. connect GitHub repo
2. publish root directory (`.`)
3. add env vars:
   - SUPABASE_URL
   - SUPABASE_ANON_KEY
   - SUPABASE_SERVER_KEY
4. trigger fresh deploy after env var changes

Current production URLs:
- Main site: https://sarvamdentalclinic.netlify.app/
- Admin: https://sarvamdentalclinic.netlify.app/doctors-login.html
- Analytics: https://sarvamdentalclinic.netlify.app/analytics.html

Vercel
------
Compatible, but backend functions need a Vercel adapter.

To move to Vercel:
1. keep frontend as-is
2. recreate `/api/*` routes as Vercel functions
3. keep same route paths
4. add same env vars
5. retest full admin flow

Conclusion:
- Vercel is a good alternative
- but Netlify functions are not drop-in Vercel functions

Hostinger
---------
Possible only if the selected plan supports backend execution.

If Hostinger plan is static-only:
- public site works
- secure backend admin routes do not

If Hostinger supports Node or similar backend execution:
- recreate `/api/*` routes there
- keep backend key server-side only
- retest Manage Users and Reset Past Data carefully

GoDaddy
--------
Same rule as Hostinger.

If static/basic hosting only:
- public site works
- secure admin backend routes do not

If backend execution is available:
- recreate `/api/*` routes
- add env vars securely
- retest all admin flows

Portable Rule
-------------
The frontend should keep calling:
- /api/admin-create-user
- /api/admin-list-users
- /api/reset-past-data

If hosting changes, preserve those paths so frontend code does not need major changes.

Pre-Deployment Checklist
------------------------
1. Verify Supabase project URL is correct.
2. Verify anon key is correct.
3. Verify server key is correct.
4. Verify Auth URL Configuration is correct.
5. Verify dentist/staff users exist in `admin_profiles`.
6. Verify dentist TOTP works.

Supabase Auth URL Configuration
-------------------------------
Recommended production Site URL:
- https://sarvamdentalclinic.netlify.app

Recommended Redirect URLs:
- https://sarvamdentalclinic.netlify.app
- https://sarvamdentalclinic.netlify.app/doctors-login.html
- http://localhost:8888/doctors-login.html
- http://localhost:8889/doctors-login.html

Post-Deployment Smoke Test
--------------------------
Test all of these:
- public booking
- same-day expired slot hiding
- weekly off overlay
- blocked date/session behavior
- dentist login
- staff login
- TOTP enrollment/verification
- forgot password
- reset password
- approve / reject
- Book for Patient
- Day Snapshot
- Control Panel close button on mobile
- Control Panel browser Back close behavior
- Manage Users
- Current Access Users
- Back to Dashboard visibility for logged-in admins only
- analytics page access and charts

Best Practical Recommendation
-----------------------------
For this project today:
1. Netlify is best
2. Vercel is second-best with a small backend adapter
3. Hostinger/GoDaddy are acceptable only if backend execution is supported
